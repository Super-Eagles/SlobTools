# _ai_docs/ 文档结构完整说明

本文件说明由 cpp-project-doc skill 生成的每个文档的详细结构，
帮助 AI 快速定位目标内容在哪个文档的哪个部分。

---

## 文档清单与内容范围

### 00_AI_CONTEXT.md（入口文档，永远最先读）

**包含内容**：
- 项目定位与目标用户
- 技术栈全貌（语言/库/版本）
- 目录结构解读（每个目录的职责）
- 核心类速查表（类名 → 文件 → 一句话职责 → 关键方法）
- 最重要的 N 个函数（枢纽函数排名）
- 主数据流图（最核心业务场景）
- 项目约定速查（命名规范/错误处理/线程安全约定）
- 文档索引表（快速路由到其他文档）

**使用时机**：每次对话的第一步，必读。

---

### 01_PROJECT_OVERVIEW.md（项目总览）

**包含内容**：
- 功能需求描述
- 目标平台与运行环境
- 已实现功能列表
- 待实现功能列表
- 项目历史与版本记录（如有）

**使用时机**：用户问"这个项目是做什么的"、需要了解功能边界。

### 02_ARCHITECTURE.md（架构设计）

**包含内容**：
- 分层架构图（UI层/业务层/数据层/工具层）
- 模块依赖关系图（有向图，箭头表示依赖方向）
- 模块边界规则（哪些模块不应互相依赖）
- 关键设计决策及其原因
- 与常见架构模式的对比说明（MVC/MVP/MVVM等）

**使用时机**：用户问架构、模块划分、想添加新功能时确定放在哪个模块。

**内部结构**：
```
## 分层架构
## 模块列表
### 模块名
  - 职责
  - 依赖的模块
  - 被哪些模块依赖
  - 公开的接口/类
## 设计决策记录
```

### 05_CLASS_REFERENCE.md（类参考，通常最大）

**包含内容**：

每个类一个独立章节，格式：

```
### 类：`ClassName`
文件路径
继承关系（每个基类的作用）
模板参数（如有）
设计模式
线程安全性声明
Qt 元对象特性（Q_OBJECT/Q_PROPERTY等）

#### 成员变量
（完整表格：名称/类型/权限/初始值/含义/读写者/线程约束）

#### 枚举与常量
（类内定义的枚举和常量）

#### 成员函数
（每个函数：签名/参数/返回值/职责/前置条件/调用时机/副作用）

#### Signals（Qt项目）
（每个信号：触发条件/参数含义/已知连接）

#### Slots（Qt项目）
（每个槽：响应的信号/连接位置/执行线程）
```

**定位技巧**：
```bash
# 找某个类的起始行
grep -n "^### 类：\`ClassName\`" 05_CLASS_REFERENCE.md

# 找某个成员变量
grep -n "m_variableName" 05_CLASS_REFERENCE.md

# 找某个方法
grep -n "methodName()" 05_CLASS_REFERENCE.md
```

### 06_FUNCTION_REFERENCE.md（函数参考，通常最大）

**包含内容**：

每个函数一个独立章节，格式：

```
━━━ 函数：ClassName::functionName() ━━━
文件路径 + 行号范围
完整签名

逐行/逐段逻辑（含行号，每段详细解释）

关键局部变量表

错误处理路径（所有错误分支）

性能注意事项

被调用于（调用者列表）

调用了（被调用者列表）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**定位技巧**：
```bash
# 找某个函数
grep -n "函数：ClassName::methodName\|^━.*ClassName::methodName" \
  06_FUNCTION_REFERENCE.md

# 找包含某个局部变量的函数
grep -n "localVarName" 06_FUNCTION_REFERENCE.md | head -20
```

### 07_DATA_FLOW.md（数据流）

**包含内容**：

每个主要用户场景/系统事件一个章节，格式：

```
### 场景N：[场景名]
触发入口
起点函数
终点状态

完整调用链（含线程标注，用缩进表示调用层次）

数据变换过程（原始类型 → 中间类型 → 最终类型）

涉及的成员变量（场景前值 → 场景后值）
```

**典型场景列表**（根据项目类型）：
1. 应用启动与初始化
2. 用户打开文件
3. 用户触发核心处理
4. 处理进度更新
5. 处理结果显示
6. 用户保存结果
7. 错误发生与恢复
8. 应用正常退出

**定位技巧**：
```bash
# 列出所有场景
grep -n "^### 场景" 07_DATA_FLOW.md

# 找涉及某个函数的场景
grep -n "ClassName::methodName" 07_DATA_FLOW.md
```

## 文档覆盖范围说明

### 文档覆盖的内容

✅ 项目自身的所有 .h/.cpp/.ui/.qrc 文件  
✅ 构建系统（CMakeLists.txt / .pro 文件）  
✅ 自定义的宏、类型别名、命名空间  
✅ 函数的逻辑意图、设计决策、潜在问题  

### 文档不覆盖的内容

❌ Qt 框架本身的 API（如 QWidget 的所有方法）  
❌ 第三方库的内部实现  
❌ 系统头文件（<algorithm>、<vector> 等）  
❌ 文档生成后才新增的代码  

遇到上述内容，回退到直接查阅官方文档或扫描源码。
